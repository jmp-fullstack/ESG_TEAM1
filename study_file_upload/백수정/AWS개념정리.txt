EC2 : 컴퓨터 한대를 빌려 사용하는 개념, *인스턴스 생성
클라우드를 통해 서버, 스토리지, 데이터베이스 서비스를 제공 받을수있음.

보안그룹 구성 : 인바운드(들오는거), 아웃바운드(나가는거) 존재 
- 보안그룹은 합집함임 : 보안그룹 지정안할시 기본값(인바운드) 모두 차단, 나가는 트래픽(아웃바운드)는 모두 허용임
 -> 따라서 보안그룹을 따로 설정하지 않으면 실행 불가능함
 -> 추가로 인바운드 규칙으로 우리는 mariaDB포트와 웹에서 받아오는 https포트를 지정해주면됨
- 프로토콜 : 데이터 통신을 위한 규칙, 데이터 주고는받는 방법을 정의 해줘야함.
 -> 전송계층 (TCP : 신뢰성 있는 데이터 , UDP : 신뢰성 보장하지않음, 손실 발생할수있음)
 -> 포트 : http(80), node(3000) 등등 필요한것 설정해주면 됨.
 -> 응용, 네트워크 등등 프로토콜의 개념은 많지만 우리한테 보안그룹에 중요한것 전송계층만 알면 나머지는 필요할때 찾아보면됨

RDS(Relational database service) : 관계형 데이터베이스임. (DB엔진 여러가지 제공, 저희는 mariaDB를 사용)
-> RDS생성은, 버전지정,이름,크기,스토리지설정 등 생성하면 간단하기 때문에 따로 설명x

S3(simple storage service) : 클라우드 스토리지 서비스 웹에 데이터(이미지,동영상) 저장하는 저장소(ex.구글 드라이브)
버킷을 만들고 S3는 버킷에 파일을 업로드 해야함. (파일: 객체라고 불림)
-미디어 파일을 저장하는데 매우 적합한 서비스인 이유?
->높은 내구성과 가용성, 확장성, 빠른 데이터 전송, 다양한 스토리지 클래스, 간편한 관리 및 보안 등등
